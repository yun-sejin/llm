"Detected zombie job" 오류는 Airflow에서 태스크가 실행 중이지만, 실제로는 더 이상 실행되지 않는 상태를 감지했을 때 발생합니다. 이는 일반적으로 태스크가 비정상적으로 종료되었거나, 워커가 중단되었을 때 발생할 수 있습니다. 이를 해결하기 위해 다음 단계를 따를 수 있습니다.

1. 태스크에 타임아웃 설정
태스크에 타임아웃을 설정하여 일정 시간이 지나면 태스크가 자동으로 종료되도록 합니다.

2. 태스크에 재시도 설정
태스크가 실패할 경우 자동으로 재시도하도록 설정합니다.

3. 태스크에 depends_on_past 설정
이전 실행이 완료될 때까지 현재 실행을 기다리도록 설정합니다.

4. 태스크에 execution_timeout 설정
태스크가 지정된 시간 내에 완료되지 않으면 실패하도록 설정합니다.

다음은 위의 설정을 적용한 예제입니다.

수정된 코드

import pendulum
from airflow.decorators import dag, task
from common_func2 import setup_logger, failure_callback  # setup_logger와 failure_callback 임포트

logger = setup_logger(__name__)

default_args = {
    'retries': 3,  # 재시도 횟수
    'retry_delay': pendulum.duration(minutes=5),  # 재시도 간격
    'depends_on_past': False,  # 이전 실행에 의존하지 않음
    'execution_timeout': pendulum.duration(minutes=10),  # 실행 타임아웃
    'email_on_failure': True,
    'email': ['luemier@daum.net'],
    'on_failure_callback': failure_callback
}

@dag(
    schedule="*/10 * * * *",
    start_date=pendulum.datetime(2021, 1, 1, tz="UTC"),
    catchup=False,
    default_args=default_args
)
def email():
    @task
    def first_task():
        logger.info("This is the first task.")
        # pdb.set_trace()  # 디버거 설정
    
    @task
    def end_task():
        logger.info("This is the SECOND task.")
        breakpoint()  # 디버거 설정
        # Simulate task failure for demonstration purposes
        # raise ValueError("This is a simulated task failure.")

    first_task() >> end_task()

DAG = email()

설명
default_args에 retries, retry_delay, depends_on_past, execution_timeout을 설정하여 태스크의 재시도, 재시도 간격, 이전 실행 의존성, 실행 타임아웃을 설정합니다.
email.py 파일에서 common_func2 모듈을 임포트하고, failure_callback 함수를 사용합니다.
first_task와 end_task 함수에 로깅을 추가하여 태스크 실행 시 로그를 기록합니다.
이제 Airflow DAG에서 태스크가 비정상적으로 종료되거나 워커가 중단되었을 때 자동으로 재시도하고, 일정 시간이 지나면 태스크가 자동으로 종료되도록 설정되었습니다. 이를 통해 "Detected zombie job" 오류를 방지할 수 있습니다.

"Detected zombie job" 오류를 해결하기 위한 다른 방법으로는 다음과 같은 접근 방식을 고려할 수 있습니다:

1. 워커 설정 조정
Airflow 워커의 설정을 조정하여 좀비 태스크를 감지하고 처리하는 방법을 개선할 수 있습니다.

2. 태스크 상태 확인 및 재시작
태스크 상태를 주기적으로 확인하고, 좀비 상태로 감지된 태스크를 재시작하는 스크립트를 작성할 수 있습니다.

3. 워커 모니터링 및 자동 재시작
워커를 모니터링하고, 비정상적으로 종료된 워커를 자동으로 재시작하는 시스템을 구축할 수 있습니다.

4. DAG 설정 조정
DAG 설정을 조정하여 좀비 태스크를 방지할 수 있습니다.

다음은 워커 설정을 조정하고, 태스크 상태를 주기적으로 확인하는 예제입니다.

1. 워커 설정 조정
Airflow 설정 파일 (airflow.cfg)에서 다음 설정을 조정합니다:


[core]
# 워커가 좀비 태스크를 감지하는 주기 (초)
scheduler_zombie_task_threshold = 300

[celery]
# 워커가 주기적으로 상태를 보고하는 주기 (초)
worker_send_task_events = True

2. 태스크 상태 확인 및 재시작 스크립트
태스크 상태를 주기적으로 확인하고, 좀비 상태로 감지된 태스크를 재시작하는 스크립트를 작성합니다.

check_and_restart_zombie_tasks.py

import pendulum
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.models import TaskInstance
from airflow.utils.state import State
from airflow.utils.db import provide_session

@provide_session
def check_zombie_tasks(session=None):
    # 현재 실행 중인 태스크 인스턴스를 가져옵니다.
    running_tasks = session.query(TaskInstance).filter(TaskInstance.state == State.RUNNING).all()
    for task in running_tasks:
        # 태스크가 좀비 상태인지 확인합니다.
        if task.is_zombie():
            print(f"Detected zombie task: {task.task_id}")
            # 태스크를 재시작합니다.
            task.set_state(State.NONE, session=session)

def restart_zombie_tasks():
    check_zombie_tasks()

default_args = {
    'owner': 'airflow',
    'start_date': pendulum.datetime(2021, 1, 1, tz="UTC"),
    'retries': 3,
    'retry_delay': pendulum.duration(minutes=5),
}

with DAG(
    'check_and_restart_zombie_tasks',
    default_args=default_args,
    schedule_interval='*/10 * * * *',
    catchup=False,
) as dag:
    check_and_restart = PythonOperator(
        task_id='check_and_restart_zombie_tasks',
        python_callable=restart_zombie_tasks,
    )



설명
airflow.cfg 파일에서 scheduler_zombie_task_threshold와 worker_send_task_events 설정을 조정하여 좀비 태스크 감지 주기와 워커 상태 보고 주기를 설정합니다.
check_and_restart_zombie_tasks.py 스크립트를 작성하여 주기적으로 실행 중인 태스크를 확인하고, 좀비 상태로 감지된 태스크를 재시작합니다.
check_zombie_tasks 함수는 현재 실행 중인 태스크 인스턴스를 가져와 좀비 상태를 확인하고, 좀비 상태로 감지된 태스크를 재시작합니다.
restart_zombie_tasks 함수는 check_zombie_tasks 함수를 호출하여 좀비 태스크를 확인하고 재시작합니다.
DAG을 정의하여 check_and_restart_zombie_tasks 태스크를 주기적으로 실행합니다.
이 방법을 통해 좀비 태스크를 감지하고 자동으로 재시작할 수 있습니다. 이를 통해 "Detected zombie job" 오류를 방지할 수 있습니다.
