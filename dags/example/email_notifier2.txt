Airflow에서 email.html 파일을 사용하지 않고 직접 템플릿을 작성하여 이메일 알림을 보내도록 설정할 수 있습니다. 이를 위해 EmailNotifier 클래스에서 템플릿 내용을 직접 정의하고, Jinja 템플릿을 사용하여 이메일 내용을 생성할 수 있습니다. 아래는 수정된 예제입니다.

import requests
import logging
from jinja2 import Template
from airflow.utils.context import Context
from basenotifier import BaseNotifier

logger = logging.getLogger(__name__)

class EmailNotifier(BaseNotifier):
    def __init__(self, from_email: str, to: list):
        super().__init__()
        self.from_email = from_email
        self.to = to
        self.api_endpoint = "http://your-api-endpoint.com/send-email"  # API 엔드포인트를 클래스 내부에서 설정
        self.subject_template = "Task {task_id} in DAG {dag_id} {status}"  # 이메일 제목 템플릿을 클래스 내부에서 설정
        self.html_template = """
        <!DOCTYPE html>
        <html lang="en">
        <head>
            <meta charset="UTF-8">
            <title>Email Notification</title>
        </head>
        <body>
            <h3>Task {{ task_id }} in DAG {{ dag_id }} {{ status }}</h3>
            <p>Execution Date: {{ execution_date }}</p>
            <p>Log URL: <a href="{{ log_url }}">{{ log_url }}</a></p>
            <p>Try Number: {{ try_number }}</p>
        </body>
        </html>
        """  # HTML 템플릿을 클래스 내부에서 직접 정의

    def notify(self, context: Context, status: str):
        subject = self.subject_template.format(
            dag_id=context['dag'].dag_id,
            task_id=context['task_instance'].task_id,
            status=status
        )
        template = Template(self.html_template)
        html_content = template.render(
            dag_id=context['dag'].dag_id,
            task_id=context['task_instance'].task_id,
            execution_date=str(context['execution_date']),
            log_url=context['task_instance'].log_url,
            try_number=context['task_instance'].try_number,
            status=status,
            ti=context['task_instance']
        )
        message = {
            "from": self.from_email,
            "to": self.to,
            "subject": subject,
            "html_content": html_content,
            "dag_id": context['dag'].dag_id,
            "task_id": context['task_instance'].task_id,
            "execution_date": str(context['execution_date']),
            "log_url": context['task_instance'].log_url
        }
        
        response = requests.post(self.api_endpoint, json=message)
        
        if response.status_code != 200:
            logger.error(f"Failed to send email via API. Status code: {response.status_code}, Response: {response.text}")
        else:
            logger.info("Email sent successfully via API.")

    def notify_on_failure(self, context: Context):
        self.notify(context, "failed")

    def notify_on_retry(self, context: Context):
        if context['task_instance'].try_number > 1:
            self.notify(context, "retried")

 설명
EmailNotifier 클래스 정의:

BaseNotifier를 상속받아 EmailNotifier 클래스를 정의합니다.
__init__ 메서드에서 이메일 알림에 필요한 매개변수를 초기화합니다.
from_email을 dsllm-dp.sec@samsung.com으로 설정하고, to를 ["dsllm-dp.sec@samsung.com", "sejin78.yun@samsung.com"]으로 설정합니다.
api_endpoint, subject_template, html_template를 클래스 내부에서 직접 설정합니다.
html_template는 HTML 템플릿을 클래스 내부에서 직접 정의합니다.
notify 메서드에서 HTML 템플릿을 사용하여 이메일 내용을 생성하고, 이메일 제목과 내용을 설정합니다.
notify_on_failure 메서드를 추가하여 실패 시 이메일 알림을 보내는 로직을 구현합니다.
notify_on_retry 메서드를 추가하여 재실행 시 이메일 알림을 보내는 로직을 구현합니다. context['task_instance'].try_number가 1보다 큰 경우에만 이메일 알림을 보냅니다.
EmailNotifier 설정:

EmailNotifier 클래스를 인스턴스화하여 email_notifier 객체를 생성합니다.
to 매개변수에 이메일 수신자를 설정합니다.
DAG 정의:

@dag 데코레이터를 사용하여 DAG을 정의합니다.
schedule, start_date, catchup, default_args 등의 속성을 설정합니다.
default_args에 on_failure_callback과 on_retry_callback을 설정하여 태스크 실패 시와 재실행 시 각각 email_notifier.notify_on_failure와 email_notifier.notify_on_retry 메서드를 호출하도록 설정합니다.
태스크 정의:

@task 데코레이터를 사용하여 check_upload_delete, get_job_id_list, dedup, parse 태스크를 정의합니다.
check_upload_delete 태스크에서 AirflowSkipException을 발생시켜 태스크를 무조건 skip 상태로 처리합니다.
태스크 의존성 설정:

check_upload_delete 태스크가 get_job_id_list, dedup, parse 태스크를 순차적으로 실행하도록 의존성을 설정합니다.
이 코드는 BaseNotifier를 상속받아 EmailNotifier 클래스를 정의하고, email.html 파일을 사용하지 않고 직접 템플릿을 작성하여 실패와 재실행 시 다른 이메일 내용을 보내는 예제입니다. 이를 통해 특정 태스크가 실패하거나 재실행될 때 각각 다른 이메일 알림을 보내는 로직을 구현할 수 있습니다.
